# Guide Utilisateur Standard - FinTrack Pro

## Vue d'ensemble

En tant qu'utilisateur standard de FinTrack Pro, votre rôle principal est la saisie quotidienne des transactions financières. Vous contribuez à la traçabilité financière de l'entreprise en enregistrant précisément les recettes et dépenses de vos activités.

## Permissions et Limitations

### Droits d'Accès

- **Saisie Transactions** : Créer et modifier vos propres transactions
- **Consultation Limitée** : Voir uniquement les activités qui vous sont assignées
- **Modification Restreinte** : Vos transactions uniquement (avant approbation)
- **Pas de Validation** : Approbation nécessite un Agent ou Administrateur

### Responsabilités

- Saisie précise et complète des transactions
- Fournir justifications claires pour chaque opération
- Respecter les délais de saisie (idéalement le jour même)
- Corriger rapidement les erreurs signalées

## Accès aux Activités

### Mes Activités Assignées

1. **Visualisation**
   - Dashboard principal → Vos activités assignées
   - Liste avec icône, nom, et mini-KPIs

2. **Droits par Activité**
   - **Activités assignées** : Plein accès saisie
   - **Autres activités** : Consultation uniquement (si autorisé)

3. **États d'Activités**
   - **Active** : Saisie normale possible
   - **Suspendue** : Saisie bloquée temporairement
   - **Fermée** : Consultation historique uniquement

## Saisie des Transactions

### Interface de Saisie

#### Accès à la Saisie

1. **Depuis le Dashboard**
   - Clic sur une activité → "Nouvelle Transaction"
   - Ou bouton flottant "+" en bas à droite

2. **Depuis le Menu**
   - Menu → Transactions → "Nouvelle Transaction"
   - Sélection de l'activité dans la liste déroulante

#### Formulaire de Saisie

**Informations Obligatoires :**
- **Activité** : Pré-sélectionnée ou choisie
- **Type** : Recette ou Dépense (boutons radio)
- **Montant** : Format décimal (ex: 1234.56)
- **Description** : Justification claire et précise
- **Date** : Date de l'opération (calendrier)

**Informations Optionnelles :**
- **Documents** : Justificatifs (factures, reçus)
- **Catégorie** : Classification complémentaire
- **Commentaires** : Notes internes

### Bonnes Pratiques de Saisie

#### Règles de Base

- **Précision** : Vérifier le montant deux fois
- **Clarté** : Description permettant identification facile
- **Complétude** : Tous champs obligatoires remplis
- **Actualité** : Saisir le jour de l'opération

#### Exemples de Descriptions

**Recettes :**
- ✅ "Vente produits cosmétiques - Client Dupont - Facture F2025-001"
- ✅ "Paiement loyer commercial - Propriétaire Martin - Chèque #1234"
- ❌ "Vente" (trop vague)

**Dépenses :**
- ✅ "Achat fournitures bureau - Papeterie Central - Ticket 456"
- ✅ "Réparation véhicule - Garage AutoPlus - Devis D2025-078"
- ❌ "Achats" (insuffisant)

### Gestion des Documents

#### Upload de Justificatifs

1. **Formats Acceptés**
   - Images : JPG, PNG, PDF
   - Taille max : 10 Mo par fichier
   - Multiples fichiers par transaction

2. **Procédure**
   - Clic "Ajouter Document"
   - Sélection fichiers ou drag & drop
   - Aperçu avant validation

3. **Bonnes Pratiques**
   - Nommer clairement les fichiers
   - Scanner/nettoyer les documents
   - Vérifier lisibilité

#### Stockage et Accès

- Documents stockés localement chiffrés
- Accès via historique transaction
- Téléchargement possible pour consultation

## Suivi de Mes Transactions

### Historique Personnel

#### Consultation

1. **Accès** : Menu → Transactions → "Mes Transactions"
2. **Filtres Disponibles** :
   - Par activité
   - Par statut (Brouillon, En attente, Approuvé, Rejeté)
   - Par période
   - Par montant

#### Statuts des Transactions

- **Brouillon** : En cours de saisie (pas soumise)
- **En attente** : Soumise, attente approbation
- **Approuvé** : Validée, comptabilisée
- **Rejeté** : Refusée avec motif

### Modification de Transactions

#### Avant Soumission

- **Brouillon** : Modification libre jusqu'à soumission
- **Actions** : Modifier, Supprimer, Soumettre

#### Après Soumission

- **En attente** : Modification impossible (contacter Agent)
- **Rejetée** : Possibilité correction et resoumission
- **Approuvée** : Modification exceptionnelle (audit requis)

#### Procédure de Correction

1. **Transaction Rejetée**
   - Notification reçue avec motif de rejet
   - Clic "Modifier" sur la transaction
   - Corriger les erreurs signalées
   - Resoumettre pour approbation

2. **Demande Exceptionnelle**
   - Contacter Agent ou Administrateur
   - Justifier la modification nécessaire
   - Attendre autorisation

## Comprendre les KPIs

### Indicateurs Personnels

#### Sur le Dashboard

- **Mes Transactions du Jour** : Nombre saisies aujourd'hui
- **En Attente d'Approbation** : Transactions soumises non validées
- **Taux d'Approbation** : Pourcentage validations sur total
- **Temps Moyen Approbation** : Délai moyen de validation

#### Par Activité

- **Recettes Saisies** : Total vos recettes (tous statuts)
- **Dépenses Saisies** : Total vos dépenses (tous statuts)
- **Solde Contribution** : Recettes - Dépenses validées

### Interprétation

#### Couleurs et Tendances

- **Vert** : Performance positive, objectifs atteints
- **Orange** : Attention requise, légers retards
- **Rouge** : Action nécessaire, problèmes détectés

#### Périodes de Référence

- **Aujourd'hui** : Activité immédiate
- **Cette semaine** : Tendance courte
- **Ce mois** : Performance mensuelle
- **Cumulé** : Contribution globale

## Bonnes Pratiques Utilisateur

### Organisation Quotidienne

#### Routine de Saisie

- **Matin** : Saisir transactions de la veille
- **Journée** : Saisir opérations courantes
- **Fin journée** : Vérifier saisies, corriger rejets

#### Planning

- **Ponctualité** : Saisir le jour même
- **Qualité** : Prendre le temps nécessaire
- **Vérification** : Relire avant soumission

### Qualité des Données

#### Éviter les Erreurs Courantes

- **Montants** : Vérifier décimales, zéros inutiles
- **Dates** : Date réelle de l'opération
- **Descriptions** : Assez détaillées pour identification
- **Types** : Recette = entrée d'argent, Dépense = sortie

#### Auto-contrôle

- **Cohérence** : Montant logique par rapport à l'activité
- **Doublons** : Vérifier pas de saisie multiple
- **Justificatifs** : Documents probants fournis
- **Orthographe** : Descriptions lisibles et correctes

### Gestion des Rejets

#### Analyse des Motifs

- **"Montant incohérent"** : Vérifier calcul, décimales
- **"Justification insuffisante"** : Détailler davantage
- **"Documents manquants"** : Fournir justificatifs
- **"Date incorrecte"** : Corriger la date

#### Prévention

- **Modèles** : Créer des descriptions types
- **Vérifications** : Double contrôle avant soumission
- **Formation** : Comprendre les attentes des validateurs
